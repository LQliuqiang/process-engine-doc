流程引擎接口文档

1.部署工作流模型

  1. url: /process/deployProcessModel
  2. 请求方式: post
  3. 支持格式: json
  4. 请求参数:
    参数 必选 类型 说明
    processDefinitionKey 是 字符串 工作流模型标识的键（用户自定义）
    name 是 字符串 工作流模型名称
    documentation 否 字符串 工作流模型描述
    events 是 列表-对象
    参数 必选 类型 说明
    id 是 字符串 id号
    name 是 字符串 名称
    type 是 数字 选择事件类型(0:开始事件 1:结束事件)
    documentation 否 字符串 描述
    		事件节点
    userTasks 是 列表-对象
    参数 必选 类型 说明
    id 是 字符串 id号
    taskName 是 字符串 任务名称
    userId 是 字符串 用户id
    documentation 是 字符串 单据内容
    		用户任务节点
    gateways 否 列表-对象
    参数 必选 类型 说明
    id 是 字符串 id号
    name 是 字符串 名称
    type 是 数字 网关类型(0:排他网关 1:并行网关 2:包容网关 3:事件网关 其它number:复杂网关)
    documentation 否 字符串 描述
    		网关节点
    sequenceFlows 是 列表-对象
    参数 必选 类型 说明
    id 否 字符串 id号
    name 否 字符串 名称
    documentation 否 字符串 描述
    conditionExpression 否 字符串 条件表达式(如:${money>500})
    sourceRef 是 字符串 起始节点（引用 事件/用户任务/网关 的id）
    targetRef 是 字符串 目标节点（引用 事件/用户任务/网关 的id）
    		流程串接节点
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误
    data 字符串 工作流模型id
    errorInfo 字符串 错误信息提示，只有状态码为240时，此数据字段不为空

2.删除工作流模型

  1. url: /process/deleteProcessModel
  2. 请求方式: get
  3. 支持格式: json
  4. 请求参数:
    参数 必选 类型 说明
    deploymentId 是 字符串 工作流模型id
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误
    data 字符串 success
    errorInfo 字符串 错误信息提示，只有状态码为240时，此数据字段不为空

  1. url: /process/processModelList
  2. 请求方式: get
  3. 支持格式: json
  4. 请求参数: null
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误
    data 字符串 工作流模型id的列表
    errorInfo 字符串 错误信息提示，只有状态码为240时，此数据字段不为空

4.部署工作流

  1. url: /process/deployProcess
  2. 请求方式: post
  3. 支持格式: json
  4. 请求参数:
    参数 必选 类型 说明
    userId 是 字符串 用户id
    processDefinitionKey 是 字符串 工作流模型标识的键（用户自定义）
    processInstanceName 是 字符串 工作流名称（用户自定义）
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误
    data 工作流实例对象
    字段 说明
    id 工作流实例id号
    name 工作流实例名称
    processTypeName 工作流模型的名称
    processDefinitionKey 工作流模型标识的键
    startTime 工作流实例开启的时间
    deploymentId 工作流模型id
    userId 用户id
    errorInfo 字符串 错误信息提示，只有状态码为240时，此数据字段不为空

7.重置工作流

  1. url: /process/resetProcess
  2. 请求方式: get
  3. 支持格式: json
  4. 请求参数:
    参数 必选 类型 说明
    processInstanceId 是 字符串 工作流实例id号
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误

10.任务处理

  1. url: /process/taskHandle
  2. 请求方式: post
  3. 支持格式: json
  4. 请求参数:
    参数 必选 类型 说明
    processInstanceId 是 字符串 工作流实例id号
    taskId 是 字符串 任务id号
    map 否 键值对对象 如果工作流中有定义数据字段模板，此map中定义的值就是改变或赋值给这个数据字段模板
  5. 返回参数:
    字段 类型 说明
    status 数字 状态码:200访问成功;240访问出现错误
    data 字符串 success
    errorInfo 字符串 错误信息提示，只有状态码为240时，此数据字段不为空
